Help Document for KootNet Sensor Units


As of right now, a Raspberry Pi is required at the very least with the option of adding supported Sensors. Additional Sensor support can be added fairly easily so long as the Sensor supports Python3, the Raspberry Pi and you have a bit of Python knowledge. I have yet to add a section on adding your own sensor support.


To make your own Sensor, find the instructions on the Programs Website or in the Help menu of Kootnet Sensors – PC Control Center.


Once installed, all KootNet Sensor program files are located in /home/sensors/, while all KootNet Sensor custom files, such as configuration and databases, are located in the /home/pi/KootNetSensors/ folder.


Configuration & Info Files

zInstalled.txt = KootNet Sensors Install Date

LastUpdated.txt = KootNet Sensors Last Updated Date & Type

installed_sensors.txt = Enabled Sensors

config.txt = Configuration


“Clean Upgrade” Scripts

These 2x Upgrades scripts delete the program directory /home/sensors/ & scripts in /home/pi/ then re-installs the programs, while keeping the configuration and data files.


upgrade_online_clean.sh = Runs a Clean HTTP Upgrade off KootNet Servers

upgrade_smb_clean.sh = Runs a Clean SMB (Windows Share) Upgrade

** Requires Configuration in /home/sensors/update_programs_smb.sh **


SQLite3 Database Files

SensorIntervalDatabase.sqlite = Interval Recordings

SensorTriggerDatabase.sqlite = Trigger Recordings


Log Files

Operations_log.txt = Contains Configuration and Database operations and errors

Sensor_Commands_log.txt = Contains Network Command operations and errors

Sensors_log.txt = Contains Sensor reading errors


Re-Configure the Sensor

This will open and allow you to edit all the configuration files, one after another, then restart the sensor programs and print to screen the current configuration and run sensor tests. Each configuration file should have sections to describe whats its for.


Note: You can type in part of a command, then hit the ‘tab’ key on your keyboard to auto complete the command. For example, type sudo ./con then hit the ‘tab’ key to complete the line.


sudo ./config_edit.sh


Test the Sensor

This will test the currently selected sensors in your configuration. If all is working well, it will grab the readings off the sensors and print them on the screen. If somethings wrong, it will either print an error and or all 0 for the data readings.


python3 test_sensors.py


Program Auto Start

The Sensor programs will automatically restart them selves every Min (If not running) through a script run in the roots crontab. This crontab is updated every time you re-configure the sensors or run an update.


If at any time you want the programs to NOT auto start, run the following command in the terminal.


sudo crontab -r


If you change your mind, and want to re-enable the auto start, simply run the command shown in the “Re-Configure Sensor” section, it will auto adds or replaces the crontab entries.
